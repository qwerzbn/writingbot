# Notebook 模块视觉验收与彩排脚本（可执行版）

适用日期：2026-03-16 起  
适用目标：验证三栏工作台升级后，演示链路“可看、可点、可讲、可复现”。

## 1. 一键检查命令

在项目根目录执行：

```bash
bash scripts/notebook_demo_check.sh
```

该命令会依次执行：
1. `scripts/demo_readiness_check.sh`（服务与接口就绪）
2. `pytest -q tests/test_notebook_service_v2.py tests/test_notebook_router_v2.py`（Notebook 后端能力）
3. `web/e2e/notebook.spec.ts`（Notebook 前端关键交互）

通过标准：三步全部返回成功。

## 2. 视觉验收（人工点击，3-5分钟）

访问：`http://localhost:3000/notebook`

### Step A：三栏结构与信息密度

操作：
1. 打开 Notebook 页面。
2. 观察左中右三栏是否同时可见。

期望：
1. 左栏显示“资料与笔记本”，含“新建”“知识库选择”“生成”按钮。
2. 中栏显示“笔记列表”，含搜索和标签筛选。
3. 右栏显示“笔记编辑器 + 关系与洞察”。
4. 顶部 Header 显示当前笔记本名、自动候选开关、`迁移 records`。

### Step B：折叠交互（演示亮点）

操作：
1. 点击左栏右上角折叠按钮。
2. 再次点击展开。
3. 对中栏重复一次。

期望：
1. 左/中栏可收起为窄竖条（显示“资料/笔记”竖排标签）。
2. 展开后内容恢复，编辑区宽度自动回流。

### Step C：导入任务与首卡打开

操作：
1. 左栏选择知识库。
2. 点击“生成”。
3. 观察任务卡片状态从 `running` 到 `done`。
4. 点击“打开首个卡片”。

期望：
1. 任务进度条变化，状态文本可见。
2. 成功后可直接定位到导入卡片并进入编辑区。

### Step D：洞察面板切换

操作：
1. 在右侧“关系与洞察”切换 `知识网络` 与 `学习进度`。
2. 点击“刷新”。

期望：
1. `知识网络` 显示笔记数、概念数、关系边。
2. `学习进度` 显示覆盖论文、覆盖率、平均掌握度、薄弱主题。
3. 刷新后数据不报错，面板正常重绘。

### Step E：重跑提炼与编辑器连通

操作：
1. 点击“重跑提炼”。
2. 在编辑区修改标题/正文，等待自动保存。

期望：
1. 重跑提炼可触发且不报错。
2. 编辑器自动保存生效（无冲突时状态稳定）。

### Step F：移动端可用性

操作：
1. 浏览器切换到手机尺寸（如 iPhone 12）。
2. 切换 `资料/笔记/编辑` 三个页签。

期望：
1. 三个页签都可进入。
2. 每个页签内容完整可操作，无布局溢出。

## 3. 答辩话术挂钩（Notebook 90秒）

1. “这里是我重构后的三栏工作台，不再是旧 records 列表。”
2. “左侧是资料与导入任务，中间是可筛选笔记，右侧是编辑与知识洞察。”
3. “折叠列能进入专注编辑模式，适合答辩现场展示效率提升。”
4. “导入任务、关系网络、掌握度面板形成了科研沉淀闭环。”

## 4. 失败定位速查

1. 页面空白或组件错位：先执行 `npm run build` 看是否有前端编译错误。
2. 导入任务不动：检查 `/api/notebooks/{id}/imports/{job_id}` 是否返回状态。
3. 洞察面板无数据：检查 `/api/notebooks/{id}/graph`、`/insights` 响应。
4. 自动保存冲突：观察是否触发 409（乐观锁语义，属预期保护）。
