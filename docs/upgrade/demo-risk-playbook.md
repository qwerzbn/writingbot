# 毕设演示应急预案（在线API场景）

## 1. 模型响应慢

- 触发信号：聊天或研究流式长时间无增量输出。
- 即时动作：
  1. 口述“当前已进入多智能体执行阶段，正在等待高质量综合结果”。
  2. 立即切到已完成会话展示已有引用结果。
  3. 再回到当前任务收尾。
- 评委说明：
  - 在线模型存在抖动，系统已通过会话持久化保证演示连续性。

## 2. 网络波动或 API 短时失败

- 触发信号：页面提示请求错误或流式中断。
- 即时动作：
  1. 直接切到笔记本页。
  2. 展示已生成卡片、知识网络、学习进度。
  3. 强调“系统价值不仅在一次生成，还在知识沉淀与复用”。
- 评委说明：
  - 云端模型波动不影响已沉淀成果与本地功能链路。

## 3. 协同写作模块未连通

- 触发信号：协同写作页出现连接错误状态。
- 即时动作：
  1. 展示错误页中的“重试连接”按钮。
  2. 说明协同写作为独立子服务，主站问答/研究/笔记全功能可独立运行。
- 评委说明：
  - 系统采用模块化解耦，子服务异常不会拖垮核心链路。

## 4. 评委要求“真实性证明”

- 即时动作：
  1. 打开 `http://localhost:5001/docs`。
  2. 现场调用任一接口（推荐）：
     - `GET /api/notebooks`
     - `GET /api/notebooks/{id}/graph`
     - `POST /api/research/stream`
  3. 展示响应数据与页面结果一致。
- 评委说明：
  - 前端展示与后端 API 一一对应，支持现场验证。

## 5. 最差情况兜底顺序（30秒内恢复节奏）

1. 聊天页历史结果（有引用）。  
2. 研究页历史报告（有计划+报告）。  
3. 笔记本页（网络+进度）。  
4. 协同写作页（状态说明）。  

> 原则：先保证“价值表达”不断线，再恢复“在线生成演示”。
