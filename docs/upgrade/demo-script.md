# 毕设答辩演示台本（8分钟）

适用场景：在线 API 优先，强调多智能体架构创新。  
建议分辨率：1920x1080。  
建议窗口布局：浏览器单窗口 + 终端备用。

## 演示前 10 分钟准备

1. 启动系统：`bash start_dev.sh`。
2. 打开地址并预热：
   - 主站：`http://localhost:3000`
   - API 文档：`http://localhost:5001/docs`
   - FastWrite：`http://localhost:3002`
3. 确认有至少 1 个知识库和 1 个笔记本（避免现场新建大数据）。
4. 在聊天页先试跑 1 条问题，确保在线 API 可返回含引用结果。
5. 保留 1 个已完成研究报告会话作为应急兜底。

## 8分钟逐段脚本（操作 + 话术）

### 00:00-00:40 开场与架构定位

- 操作：
  1. 打开首页后直接进入「聊天」页。
- 话术：
  - “我的系统不是单轮问答工具，核心是 Orchestrator，把复杂科研任务拆成多角色 Agent 协作执行：规划、检索、对比、校验、写作。”

### 00:40-01:20 知识库状态确认

- 操作：
  1. 切到「知识库」页面，展示论文数量与文件列表。
  2. 不现场上传大 PDF。
- 话术：
  - “论文已完成解析、分块和向量化，后续回答都可追溯到来源证据。”

### 01:20-03:00 聊天页演示多智能体过程

- 操作：
  1. 回到聊天页并选中目标 KB。
  2. 输入：`请比较该知识库论文的方法差异、实验优势与局限，并给出来源编号。`
  3. 展示“智能体思考过程”进度条与阶段卡片。
  4. 展示回答中的引用卡片。
- 话术：
  - “这里是状态机驱动的多阶段执行，不是黑箱一次生成。”
  - “每段结论都能回到来源证据，保证可解释性和可复核性。”

### 03:00-03:50 图表理解亮点（新增）

- 操作：
  1. 在同一知识库继续提问：`请解释图3的核心结论，并说明它支持了作者哪条主张。`
  2. 展示回答下方的图表证据卡（缩略图、caption、解释摘要）。
  3. 点击证据卡，跳转 PDF 对应页。
- 话术：
  - “这里不再只理解正文段落，系统会把论文里的 Figure/Table 也抽取成可检索资产。”
  - “当用户问图3、表2、趋势或消融时，会走图文联合证据链，而不是只靠语言模型猜。”

### 03:50-04:20 从聊天沉淀到笔记

- 操作：
  1. 在助手回答右下角点击“保存为笔记”。
  2. 选择目标笔记本并确认成功。
- 话术：
  - “知识不会停留在会话窗口，会沉淀到结构化笔记中，形成可复用资产。”

### 04:20-05:50 研究页演示长链路

- 操作：
  1. 进入研究页。
  2. 输入主题：`请针对当前知识库给出研究现状、方法对比、风险与后续方向。`
  3. 展示流式“研究计划 + 研究报告”。
  4. 点击“保存为笔记”。
- 话术：
  - “研究模式体现了从规划到综合的完整多智能体链路，输出可直接进入后续写作。”

### 05:50-07:10 笔记本三栏工作台（创新点）

- 操作：
  1. 进入笔记本页，展示三栏布局。
  2. 点击“从知识库生成”，展示导入任务进度。
  3. 打开生成卡片，展示证据视图。
  4. 切换“知识网络”和“学习进度”面板。
  5. 点击“重跑提炼”。
- 话术：
  - “这里不是普通 Markdown，而是自动提炼、跨文档关联、掌握度评估的学习中枢。”

### 07:10-07:40 协同写作收口

- 操作：
  1. 打开协同写作页，展示 FastWrite 连接状态 ready。
- 话术：
  - “前面研究与笔记的结构化内容，最终进入论文写作闭环。”

### 07:40-08:00 结尾三句话

- 话术：
  1. “系统把检索、分析、沉淀、写作串成了单一科研工作流。”
  2. “核心创新在多智能体调度、图文联合证据抽取与可追溯引用机制。”
  3. “笔记本模块实现自动提炼与知识网络化，支撑长期科研积累。”

## 演示中要点名的接口（架构导向）

1. 编排：`/api/orchestrator/run`、`/api/orchestrator/stream/{run_id}`。
2. 研究流：`/api/research/stream`。
3. 笔记增强：`/api/notebooks/{id}/imports/kb`、`/api/notebooks/{id}/graph`、`/api/notebooks/{id}/insights`、`/api/notebooks/{id}/notes/{note_id}/extract`。
4. 图表资产：`/api/kbs/{kb_id}/assets`、`/api/kbs/{kb_id}/assets/{asset_id}/interpret`。
5. 主站功能页：聊天、研究、笔记本、协同写作。
